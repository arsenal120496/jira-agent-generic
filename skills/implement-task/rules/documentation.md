# Documentation rules

Issue type: Documentation. Candidate labels: `docs`, `documentation`.

## Instructions
- Update only the documentation the AC names (README, wiki page, API/code comments).
- Keep content accurate to the current code; verify claims against the source before writing.

## Rules / watch-outs
- Markdown files: include a Table of Contents, consistent heading hierarchy, no decorative
  separators or empty break lines between sections.
- ASCII only - no emojis, no AI-gen typographic characters (em/en dash -> `-`, arrow -> `->`,
  ellipsis -> `...`); `§` is allowed for legal citations.
- Do not change product code in a Documentation ticket; if code must change, go to `on_block`.
- No secrets or sensitive data (credentials, tokens, personal data) in examples/snippets.
